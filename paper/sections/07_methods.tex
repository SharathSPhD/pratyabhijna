\section{Methods}\label{sec:methods}

\subsection{Arms (v0.2 four-arm matrix)}

v0.2 redesigns the arm matrix to make the cascade's contribution \emph{substrate-matched} (the v0.1 design conflated cascade contribution with substrate gap). For each item across the four task domains we now run up to four arms:
\begin{enumerate}[leftmargin=1.4em]
\item \textbf{\texttt{haiku\_bare} (no PCE).} Anthropic Claude Haiku invoked through the Claude Code CLI in \texttt{-p} non-interactive mode with \texttt{--output-format json}, wrapped by \texttt{HaikuLM} (\S\ref{sec:engine}) so it shares the same retry, cost-ledger, and audit-log pipeline as the cascade. This is the v0.2 \emph{primary control}: the apples-to-apples baseline against which the Haiku-substrate cascade is measured.
\item \textbf{\texttt{haiku\_cascade} (with PCE).} The same Claude Haiku endpoint through the full two-pass-always cascade (draft $\to$ \iast{vimar\'sa} brief $\to$ revision) with $K{=}3$ candidates per pass and \texttt{max\_tokens}$=150$. This is the v0.2 \emph{primary treatment}.
\item \textbf{\texttt{local\_bare} (no PCE).} Qwen2-1.5B-Instruct invoked with $\textsc{Cit}$ alone (single sample, no cascade). Acts as a substrate-strength baseline so absolute scores can be compared across substrates.
\item \textbf{\texttt{local\_cascade} (with PCE).} The same local LLM through the full cascade. Reported as a within-substrate ablation when scope permits; in the v0.2 pilot this arm was deferred to v0.3 due to throughput on the pilot host.
\end{enumerate}
The \emph{primary v0.2 contrast} is \texttt{haiku\_cascade} vs.\ \texttt{haiku\_bare}; the substrate-baseline contrast is \texttt{haiku\_bare} vs.\ \texttt{local\_bare}; the local ablation contrast is \texttt{local\_cascade} vs.\ \texttt{local\_bare}. Both cascade arms route through the same \texttt{LMProtocol} (ADR-001 / ADR-004) so the only thing that varies between cascade arms is the underlying generative substrate. All arms are scored by the \emph{identical} domain-specific scoring function (\S\ref{sec:benchmarks}) so contrasts are between observable outputs, not subjective evaluation policies, and \texttt{iccha} runs in \texttt{prompt\_mode="verbatim"} + \texttt{sampler\_grid\_mode="parity"} (ADR-005) so the bare and cascade arms see prompt- and sampler-identical conditions inside the cascade's first pass.

\subsection{Pre-registered hypotheses}

Six hypotheses are pre-registered in \texttt{docs/SPEC\_v0.2.md} \S3 \emph{before} the v0.2 pilot is run; H1--H4 are restated against the v0.2 primary control (\texttt{haiku\_bare}) but the inferential machinery is unchanged from v0.1:

\begin{itemize}[leftmargin=1.4em]
\item \textbf{H1.} \texttt{haiku\_cascade} achieves higher CreativityPrism aggregate (Quality$\cdot$Novelty$\cdot$Diversity blend) than \texttt{haiku\_bare} on the AUT slice.
\item \textbf{H2.} \texttt{haiku\_cascade} achieves higher aspect-multiplicity score than \texttt{haiku\_bare} on the Wittgenstein aspect-shift poetry-interpretation probe.
\item \textbf{H3.} \texttt{haiku\_cascade} achieves higher POEMetric advanced-creative-abilities composite than \texttt{haiku\_bare} on the poetry-generation slice.
\item \textbf{H4.} \texttt{haiku\_cascade} achieves higher creative-novelty score than \texttt{haiku\_bare} on the BBH-style scientific-creativity probes.
\item \textbf{H5.} The aggregate composite $C_0 = \tfrac{1}{2} z(H_1) + \tfrac{1}{6} z(H_2) + \tfrac{1}{6} z(H_3) + \tfrac{1}{6} z(H_4)$ (z-scaled per domain) is positive.
\item \textbf{H6.} Within-cascade, the trials in which \iast{vimar\'sa} fired (an aspect-shift event was detected on the revised surface) score higher on the H2/H3/H4 composites than the trials in which it did not. Reported separately for each cascade arm; the primary v0.2 reading is \texttt{H6\_haiku\_cascade}.
\end{itemize}
Two additional v0.2 hypotheses are introduced (see \texttt{SPEC\_v0.2.md} \S3) but are not powered by the pilot and are left for v0.3: H7 (substrate-portability: $\Delta_{\texttt{haiku\_cascade}-\texttt{haiku\_bare}} \approx \Delta_{\texttt{local\_cascade}-\texttt{local\_bare}}$) and H8 (revision-causality: revised surfaces differ from drafts in $\geq 80\%$ of cascade items, $\geq 60\%$ on a textual-similarity floor).

\subsection{Statistical protocol}

For each hypothesis we report: paired mean $\Delta$, Hedges' $g$ with small-sample correction, paired permutation $p$ (one-sided directional, exact for $n\le 18$, Monte-Carlo with $50000$ permutations otherwise), Wilcoxon signed-rank $p$ (one-sided), 95\% BCa bootstrap CI of the paired mean ($10000$ resamples), Holm--Bonferroni adjusted $p$ across $\{H_1, H_2, H_3, H_4\}$, and a-priori (assumed $g=0.5$) and retrospective (observed $g$) statistical power. A hypothesis is \emph{supported} iff its Holm-adjusted $p<0.05$ \emph{and} its BCa CI is strictly positive; we additionally call out hypotheses with strictly positive BCa lower bounds as \emph{directionally positive} when reporting pilot-scale results to make the power constraint explicit. H5 is reported as a single test on the pooled z-blend; H6 is an independent-samples Mann--Whitney $U$ on within-cascade fired-vs-not trials, computed once per cascade arm. Sensitivity analyses repeat every primary contrast against the \texttt{local\_bare} arm; the v0.2 stats payload (\texttt{benchmarks/results\_v2/stats.json}) reports three named contrasts side-by-side: \texttt{primary} (\texttt{haiku\_cascade} vs \texttt{haiku\_bare}), \texttt{local\_ablation} (\texttt{local\_cascade} vs \texttt{local\_bare}, empty in the pilot), and \texttt{substrate\_baseline} (\texttt{haiku\_bare} vs \texttt{local\_bare}).

\subsection{Reproducibility}

All seeds are recorded; bootstrap and permutation RNGs are seeded by \texttt{--seed} (default 4242). The benchmark driver (\texttt{benchmarks/driver.py}) writes a checkpoint after every call so a re-run resumes exactly from the failure point. The v0.1 audit log lives under \texttt{audit/phase9/}; the v0.2 audit log lives under \texttt{audit/} (with \texttt{audit/haiku/} for per-Haiku-call records and \texttt{audit/cost\_ledger.json} as the persistent cost ledger). Every numerical claim in this paper is generated from \texttt{benchmarks/results\_v2/stats.json} via \texttt{benchmarks/autoreport.py} and is therefore traceable end-to-end.

\subsection{TRIZ-resolved operator changes (v0.2)}

The v0.1 adversarial review (\texttt{docs/reviews/2026-04-28-adversarial-plugin-review.md}) identified five operator-level contradictions whose unresolved interactions explained the v0.1 directional null. v0.2 resolves each via TRIZ contradiction analysis (\texttt{docs/triz/}) and a corresponding ADR (\texttt{docs/adr/v0.2/}):
\begin{enumerate}[leftmargin=1.4em]
  \item \textbf{C1 / ADR-001 (cost-vs-quality of $K$ Haiku calls).} TRIZ-39 vs TRIZ-19 $\to$ principles 10/35/19 $\to$ parallel batching of the $K$ candidate calls, with per-domain $K$ deferred to v0.3.
  \item \textbf{C2 / ADR-002 (coverage-vs-novelty in \iast{apohana}).} TRIZ-27 vs TRIZ-35 $\to$ principles 13/24/8 $\to$ remove the \texttt{np.clip(apoha, 0.0, None)} in \iast{j\~n\=ana} and use a min-max-shifted \texttt{\_shift\_apoha}; signed \iast{apohana} now actively penalises must-avoid neighborhoods.
  \item \textbf{C3 / ADR-003 (reflection-vs-speed of two-pass \iast{vimar\'sa}).} TRIZ-27 vs TRIZ-9 $\to$ principles 10/11/21 $\to$ two-pass-always cascade with revision capped at one extra call; \texttt{vimarsa.min\_evidence\_points=1} so the operator can fire on a one-shot trajectory.
  \item \textbf{C4 / ADR-004 (substrate-strength-vs-cascade-overhead).} TRIZ-21 vs TRIZ-36 $\to$ principles 17/20/30/34 $\to$ a pluggable \texttt{LMProtocol} so the cascade can be measured against any substrate; the cascade contributes an aspect-shift multiplicity dimension that bare cannot.
  \item \textbf{C5 / ADR-005 (determinism-vs-creativity sampler asymmetry).} TRIZ-28 vs TRIZ-35 $\to$ principles 3/13/2 $\to$ \texttt{iccha} defaults to \texttt{prompt\_mode="verbatim"} and \texttt{sampler\_grid\_mode="parity"} inside the cascade so the bare and cascade arms cannot be told apart on prompt or sampler.
\end{enumerate}
Each ADR is end-to-end traceable: TRIZ card $\to$ ADR $\to$ operator code $\to$ unit test $\to$ pilot result.
