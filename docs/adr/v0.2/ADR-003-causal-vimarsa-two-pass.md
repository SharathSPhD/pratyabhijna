# ADR-003 (v0.2) — Causal `vimarsa` via two-pass-always cascade

Status: Accepted (frozen during planning round 1; user explicitly chose two-pass-always over single-shot revision, single-shot re-cascade, and loop-until-fire).
Date: 2026-04-28.
Related TRIZ card: [docs/triz/C3-reflection-vs-speed.md](../../triz/C3-reflection-vs-speed.md).

## Context

The adversarial review found:

- P0-2: `vimarsa` cannot fire in the v0.1 cascade because the trajectory has only one point and `switching >= 2` is required.
- P0-3: even if `vimarsa` fires, `state.surface` is not revised — `vimarsa` is post-hoc telemetry, not a generative loop.
- P2-1: poetry_gen and aut pass `aspects=[]` so `aspect_count=0` and the gate is structurally closed.

These are the central reasons v0.1 trailed bare. To fix them, the cascade must (a) actually run a revision step, and (b) make the revision causally depend on what `vimarsa` found missing.

## Decision

Replace `run_cascade`'s single-pass output with a *two-pass-always* loop:

```python
def run_cascade(
    prompt: str,
    constraint: Constraint,
    *,
    lm: LMProtocol,
    embed: Embedder,
    K: int = 4,
    cit_temperature: float = 0.9,
    max_tokens: int = 200,
    base_seed: int = 0,
    retrieval_set: list[str] | None = None,
    aspects: list[str] | None = None,
    bypass_vimarsa: bool = False,
) -> CascadeState:

    # Pass 1 — draft
    cands_d = iccha(prompt, constraint, lm=lm, K=K, base_seed=base_seed,
                    max_tokens=max_tokens, prompt_mode="verbatim", sampler_grid_mode="parity")
    apoha_d = apohana(cands_d, constraint, embed=embed, normalize=True)
    anan_d  = np.array([ananda(c, constraint=constraint, embed=embed) for c in cands_d])
    sel_d, dF_d, post_d = jnana(cands_d, apoha_d, anan_d)
    draft = kriya(cands_d[sel_d])

    # vimarsa with min_evidence_points=1, returning a brief
    event_d, novelty_d, diag_d, brief = vimarsa(
        prompt, draft, embed=embed,
        retrieval_set=retrieval_set or [], aspects=aspects or [],
        ananda_score=float(anan_d[sel_d]),
        iccha_apoha_trajectory=None,            # explicit: no pseudo-trajectory
        min_evidence_points=1,
        return_brief=True,
    )

    if bypass_vimarsa:
        return _state(prompt, constraint, draft, event_d, ..., bypass=True)

    # Pass 2 — revision conditioned on the brief (always runs)
    revision_prompt = f"{prompt}\n\nReviser brief: {brief}"
    cands_r = iccha(revision_prompt, constraint, lm=lm, K=K, base_seed=base_seed + 17,
                    max_tokens=max_tokens, prompt_mode="verbatim", sampler_grid_mode="parity")
    apoha_r = apohana(cands_r, constraint, embed=embed, normalize=True)
    anan_r  = np.array([ananda(c, constraint=constraint, embed=embed) for c in cands_r])
    sel_r, dF_r, post_r = jnana(cands_r, apoha_r, anan_r)
    revision = kriya(cands_r[sel_r])

    event_r, novelty_r, diag_r, _ = vimarsa(
        prompt, revision, embed=embed,
        retrieval_set=retrieval_set or [], aspects=aspects or [],
        ananda_score=float(anan_r[sel_r]),
        iccha_apoha_trajectory=None,
        min_evidence_points=1, return_brief=False,
    )

    return CascadeState(..., surface=revision, surface_draft=draft,
                        surface_revision=revision,
                        vimarsa_event=event_r, vimarsa_event_draft=event_d,
                        ...)
```

Key changes:

1. `state.surface = revision` (not `draft`). The revision is the cascade's output that the benchmark scores. The draft is preserved for H8.v2 diagnostics.
2. `bypass_vimarsa=True` flips the cascade back to single-pass and returns the draft as `state.surface`. This is the explicit ablation control.
3. Both passes are *always* run (no early exit on `event_d=True`). The user's frozen choice was two-pass-always specifically to make the revision-vs-draft delta measurable on every item.
4. The brief is generated by `vimarsa` listing missing aspects and is fed verbatim into the revision prompt.
5. The two passes can be executed in parallel via `asyncio.gather` when the substrate is `HaikuLM` (see ADR-001 / TRIZ C3). For `LocalLM` the two passes run sequentially because Qwen2-1.5B saturates the GPU/MPS.
6. `vimarsa.min_evidence_points` defaults to 1; with `iccha_apoha_trajectory=None` the switching gate is treated as N/A (not failed). This fixes P0-2 / P2-1 directly.

## Consequences

Positive:

- The cascade is causally self-revising; `vimarsa` directly affects the surface text via the brief.
- H8.v2 (revision_score - draft_score) becomes a clean within-treatment causal contribution test.
- Bypass flag retains a clean ablation control.
- The `min_evidence_points=1` change unblocks `vimarsa` for one-shot cascades.

Negative:

- Cost doubles relative to v0.1 (two Haiku calls per cascade item). Mitigation: pilot uses K=4 (down from v0.1's K=4-8) and runs <=22 items, well within the $15 envelope.
- Wallclock doubles relative to v0.1 unless parallelism is enabled. Mitigation: the local arms accept the sequential cost; Haiku arms use `asyncio.gather` (TRIZ C3, Principle 10).
- Brief generation on aspect-empty domains (poetry_gen, aut) cannot list "missing aspects." Mitigation: when `aspects=[]`, the brief reads "Refine for novelty, vividness, and surprise." (a generic creative-revision instruction). This keeps the revision pass useful even when the domain has no formal aspect dictionary.

## Alternatives considered

- *Single-shot revision when vimarsa misses*: rejected by user; loses ability to test H8.v2.
- *Single-shot re-cascade*: rejected by user; same loss.
- *Loop-until-fire (cap=3)*: rejected by user as expensive and divergent.
