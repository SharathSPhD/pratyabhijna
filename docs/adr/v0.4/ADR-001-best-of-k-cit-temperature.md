# ADR-001 (v0.4) — Best-of-K candidate width as `cit_temperature` mechanism

Status: Accepted (frozen at end of Phase 1).
Date: 2026-04-29.
Related TRIZ card: [docs/triz/v0.4/C2-oauth-only-vs-cit-temperature.md](../../triz/v0.4/C2-oauth-only-vs-cit-temperature.md).

## Context

In v0.3, `cit_temperature` was plumbed from `run_cascade -> iccha` and recorded on `Candidate.sampler`, but the Haiku CLI subprocess (`claude --print`) does not honour any token-level sampler flag. The optional Anthropic SDK path *did* honour `temperature` and `top_p`, but the user-imposed hard constraint forbids any `ANTHROPIC_API_KEY` path. v0.4 inherits that constraint (PRD HC-1.v4). The v0.3 adversarial review correctly flagged that `cit_temperature` was therefore "recorded intent, not causal control" for the headline Haiku benchmark.

The C2 TRIZ card resolves the contradiction via principles **10 (Preliminary action)** and **19 (Periodic action)**: replace continuous token-level temperature with a discrete K-shaped exploration whose width is controlled by `cit_temperature`.

## Decision

`iccha.generate_candidates` becomes responsible for translating `cit_temperature` into a runtime candidate count `K_runtime`:

```python
K_min, K_max = 2, 16
K_runtime = max(K_min, min(K_max, round(K_eff * (0.5 + 1.5 * cit_temperature))))
```

At `cit_temperature = 0.0`, `K_runtime = round(0.5 * K_eff)` (concentrated around the central reading). At `cit_temperature = 1.0`, `K_runtime = round(2.0 * K_eff)` (broader exploration). At `cit_temperature = 0.5`, `K_runtime ≈ K_eff` (the v0.3 default).

Each of the `K_runtime` candidates is generated by the same `claude --print` substrate but with a deterministic prompt-level diversity perturbation drawn from a frozen 8-element table:

```python
PERTURBATION_TABLE = (
    "",                                                 # 0: identity
    "Lean toward the literal interpretation. ",         # 1: literal
    "Lean toward the figurative interpretation. ",      # 2: figurative
    "Foreground the unusual aspect of the prompt. ",    # 3: foreground unusual
    "Foreground the conventional aspect of the prompt. ", # 4: foreground conventional
    "Use one specific concrete image. ",                 # 5: concrete
    "Use one abstract or universal frame. ",             # 6: abstract
    "Reframe in one alternative perspective. ",          # 7: reframe
)
```

The perturbation index for candidate `i` of an item with seed `seed` is `(seed % 8 + i) % 8`. The perturbation is prepended to the prompt as a single-line nudge; the prompt itself is unchanged. Prompts containing aspect dictionaries override the perturbation with an aspect-emphasis flip (must-have vs must-avoid) drawn from the same index space. The perturbation is recorded on `Candidate.sampler["perturbation_idx"]`.

Posterior selection inside `iccha` (parity sampler tau, jnana posterior, ananda scoring) is otherwise unchanged.

`Candidate.sampler` records:

```json
{
  "tau": 0.9,
  "cit_temperature": 0.7,
  "K_eff": 4,
  "K_runtime": 8,
  "perturbation_idx": 3
}
```

## Why this is on-theory for Pratyabhijñā

In Abhinavagupta's framework, *cit-śakti* is the breadth of awareness — the field within which possibilities are simultaneously held before any one is actualised. Modulating that breadth through *candidate-set width* is a faithful analogue of the doctrine: at high `cit_temperature` the system holds more readings of the prompt simultaneously and lets `iccha`'s posterior choose; at low `cit_temperature` it concentrates around the central reading. A token-level temperature flag would only have modulated the *micro-structure* of one continuation; this mechanism modulates the *macro-structure* of the considered set. The C2 card argues this is more on-theory than the v0.3 plumbed-but-inert flag.

## Verifiability gate

Phase 2 prove-gate (`scripts/prove_gate.py` v0.4 assertions) measures n-gram entropy across the candidate set on a 12-item probe set at `cit_temperature ∈ {0.2, 0.5, 0.9}`. The required invariant is:

> mean character-trigram entropy across candidates at `cit_temperature = 0.9` is strictly greater than at `cit_temperature = 0.2`, on the same prompts and the same base seed.

If monotonicity fails, the v0.4 paper restates `cit_temperature` as recorded-only and demotes the claim — the SPEC HC-2.v4 still holds, the prove-gate fails, and Phase 2 is not closed until either the mechanism is fixed or the paper claim is removed.

## Consequences

Positive:

- `cit_temperature` becomes causal under OAuth-only.
- The mechanism is on-theory (cit-śakti = breadth of awareness).
- Reproducible across runs (deterministic perturbation table indexed by seed).
- No new dependency, no SDK path, no API key.

Negative:

- Higher `cit_temperature` linearly increases Haiku spend per item. Mitigated by `K_max = 16` cap and Phase-2 ledger pre-flight.
- The 8-element perturbation table is a frozen design choice; v0.5 may revisit. Frozen here so v0.4 is pre-registered.

## Implementation files

- `src/pce/operators/iccha.py` — `K_runtime` computation; perturbation table; per-candidate `prompt_perturbation_idx`.
- `tests/test_cit_temperature_kruntime.py` — new test: `K_runtime(0.0) < K_runtime(0.5) < K_runtime(1.0)`; perturbation determinism.

## Acceptance gate (Phase 2)

- `tests/test_cit_temperature_kruntime.py` passes.
- Prove-gate v0.4-α reports trigram entropy ratio at `cit_temperature=0.9` vs `0.2` ≥ 1.05 on the 12-item probe set.
- `Candidate.sampler` JSON includes `cit_temperature`, `K_eff`, `K_runtime`, `perturbation_idx`.
