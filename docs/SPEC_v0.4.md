# PCE v0.4 — formal specification

Version: 0.4.0 (frozen during v0.4 planning round; supersedes [docs/SPEC_v0.3.md](SPEC_v0.3.md) for v0.4 scope).

This SPEC encodes the v0.4 mechanism study described in [docs/reviews/2026-04-29-adversarial-v0.3-review.md](reviews/2026-04-29-adversarial-v0.3-review.md). v0.3 stays as the previous-version contract; this file is the actual contract for v0.4 implementation, benchmark, and acceptance.

## 0. Purpose (delta from v0.3)

v0.3 made the substrate clean and the four-arm benchmark fair, but the pilot was directional-negative on every primary contrast. Post-hoc analysis showed the cascade *generated* better shadow revisions on 15/20 items (mean Δ = +0.0458) yet *committed* only 3 of them. Three "active-inference" mechanisms were also audit-only on the Haiku CLI path: the free-energy budget never gated the revision pass, `cit_temperature` never reached the CLI subprocess, and `vimarsa.switching_ok` was always trivially true because `iccha_apoha_trajectory` was never supplied to `vimarsa`. v0.4 fixes those gaps and re-runs as a **mechanism study** rather than a fresh superiority claim.

v0.4 carries the v0.3 hard constraints and adds two:

- **HC-1.v4 — OAuth-only.** No `ANTHROPIC_API_KEY`. `claude --print` remains the only path to Haiku.
- **HC-2.v4 — `cit_temperature` must be causally active under the OAuth-only constraint.** v0.4 implements this as best-of-K candidate width inside `iccha`; the CLI substrate is unchanged.
- **HC-3.v4 — Cost cap ≤ $30.** Powered pilot must total ≤ $30 across cascade arms, control arms, and the Sonnet judge subset.
- **HC-4.v4 — No mocks.** Same v0.2/v0.3 anti-substitution policy. If the CLI is rate-limited, surface `HaikuRateLimitError` and abort.

## 1. Scope (frozen)

In scope:

- **Substrate hardening (P0 fixes from the v0.3 review):**
  - `FreeEnergyBudget.should_continue_revision()` becomes a hard gate inside `run_cascade`. When the ledger drops below the floor before the shadow-revision pass, the cascade emits `revision_skipped_reason="fe_budget_underwater"` and commits the draft.
  - `cit_temperature` is made causal under OAuth-only via best-of-K candidate width: `iccha.generate_candidates` sets `K_runtime = clip(round(K_eff * (0.5 + 1.5 * cit_temperature)), K_min, K_max)`. Each candidate is generated by the same `claude --print` substrate but with a deterministic prompt-level diversity perturbation drawn from the seed (brief reframing or aspect-emphasis flip). The posterior selection in `iccha` is unchanged.
  - `HaikuLM._call_cli_once` parses `claude --print` stdout JSON even when `rc != 0`; raises typed `HaikuRateLimitError` when the JSON contains `"api_error_status": 429` or `result` matches the rate-limit regex; surfaces `api_error_status` and `result` into the audit trace.
  - `vimarsa.switching_ok` continues to require an `iccha_apoha_trajectory`; v0.4 makes the cascade actually pass one (the per-candidate aspect coverage from `apohana`). Items with no aspect dictionary still get `switching_ok=True` via the existing fallback.
- **Commit policy infrastructure (Experiment C):**
  - New module `src/pce/policies/commit.py` exposes a `CommitPolicy` ABC and five implementations:
    - `AlwaysDraft` — always commit draft, no shadow revision pass.
    - `AlwaysRevise` — always commit shadow revision (used by `haiku_generic_revise_2pass`).
    - `EventGated` — current v0.3 policy via `vimarsa_event`.
    - `LearnedGate` — scikit-learn `LogisticRegression` over features `[delta_F, novelty, aspect_count, ananda, budget_balance]`; trained on a held-out calibration set sampled from `audit/v0_3_traces/` (the v0.3 pilot artifacts). Leakage-controlled by leave-one-domain-out cross-validation.
    - `OracleCommit` — post-hoc only, picks the surface with the higher proxy score. Used by analysis, not as a benchmark arm.
  - `run_cascade(commit_policy=...)` accepts the new enum value `"learned_gate"` in addition to the v0.3 set.
- **Statistics rebuild (Experiments A + B + C):**
  - **H1.v4–H4.v4**: per-domain superiority of `haiku_cascade` (event_gated) vs `haiku_bare` — directly comparable to H1.v3–H4.v3.
  - **H5.v4**: meta-pool across domains; **fixed-effects** is locked (SPEC and code agree).
  - **H6.v4 / H7.v4**: matched-budget (`haiku_bare_2K_scorer`) and matched-revision (`haiku_generic_revise_2pass`) controls.
  - **H8a.v4** (Experiment A): paired `score(shadow_revision) - score(draft)` across **all** cascade items, ignoring the gate.
  - **H8b.v4** (Experiment B): event-gate calibration. `vimarsa_event` is treated as a binary classifier of "revision is better than draft" (i.e. `revision_delta > 0`). Reports precision, recall, F1, AUROC, and Brier score; reports the same metrics for `LearnedGate` as a baseline.
  - **H8c.v4** (Experiment C): commit-policy comparison. `learned_gate` vs `event_gated` vs `always_revise` vs `always_draft` on the same generated draft/revision artifacts, paired across items. `OracleCommit` is reported as a non-arm upper bound.
  - **H9.v4**: judge-proxy agreement. Spearman ρ + sign-agreement between proxy delta and Sonnet judge delta on the stratified subset.
  - All hypotheses run with paired permutation tests (`scipy.stats.permutation_test`), Hedges' g, BCa bootstrap CIs (10,000 resamples), and Holm-Bonferroni correction within the family `{H1.v4, H2.v4, H3.v4, H4.v4}` and within `{H6.v4, H7.v4}`. All JSON output strict (`allow_nan=False`).
- **Sonnet LLM-judge bridge (Experiment E, scoped down):**
  - Re-enable the existing `scripts/run_judge_bridge.py`. Frozen Sonnet judge prompt, versioned via SHA-256 hash; pairwise A/B with random position swap; ties allowed.
  - Stratified subset: 8 items / domain × 4 domains = **32 items**, drawn from quartiles of proxy delta.
  - Outputs `benchmarks/results_v0.4/judge.jsonl` and `benchmarks/results_v0.4/judge_agreement.json`.
  - Sonnet cost projected ≤ $5; bridge dry-run on 4 items required before full subset.
- **Powered pilot (Experiments A + B + C):**
  - 4 base arms (`haiku_bare`, `haiku_cascade`, `haiku_bare_2K_scorer`, `haiku_generic_revise_2pass`) × 4 domains (`poetry_gen`, `poetry_interp`, `aut`, `sci_creativity`) × **n = 20 / domain** = 320 base items.
  - For the cascade arm only, multiplex `commit_policy ∈ {event_gated, always_draft, always_revise, learned_gate}`. The same drafts and shadow revisions are reused across policies (post-hoc rescoring), so policy multiplexing adds no additional Haiku spend beyond the cascade base run.
  - Cost ledger `audit/cost_ledger_v0_4.json` records every Haiku and Sonnet call; pilot total under $30 (HC-3.v4).
- **Plugin / MCP surface:**
  - Manifest version bumped `0.3.0 → 0.4.0`.
  - `pce_cascade(...)` MCP tool gains `commit_policy` enum: `"event_gated" | "always_draft" | "always_revise" | "learned_gate"`.
  - New MCP tool `haiku_judge(arm_a: str, arm_b: str, item_id: str)` runs a single ad-hoc Sonnet pairwise judgment using the frozen prompt.
  - `scripts/smoke_plugin.py --with-haiku` adds a one-call judge round-trip and a learned-gate divergence assertion.

Out of scope (explicitly):

- No new domains beyond v0.3's four.
- No human rater study. Construct validity is local proxy + frozen Sonnet LLM-judge only.
- No SDK / API-key path. CLI substrate stays identical to v0.3 (only error parsing improves).
- No Mechanism Ablation pack (Experiment D from the review). Documented as v0.5 scope.
- No powered narrow-domain replication (Experiment F). Documented as v0.5 scope.
- No new local model substrate, no fine-tuning.
- No expansion of the score composites or scoring rubric. `benchmarks/scoring.py` is unchanged.

## 4. Operator semantics deltas

### 4.1 iccha — best-of-K width as `cit_temperature` mechanism

```python
K_eff       = K  # nominal K from cascade kwarg
K_min, K_max = 2, 16
K_runtime   = max(K_min, min(K_max, round(K_eff * (0.5 + 1.5 * cit_temperature))))
# Each candidate gets a deterministic prompt perturbation drawn from `seed`.
candidates  = [generate(prompt_perturbed_i, sampler=parity, seed=seed+i)
               for i in range(K_runtime)]
# posterior selection unchanged.
```

The perturbations are drawn from a frozen 8-element diversity table indexed by `(seed % 8, i % 8)`; perturbations are aspect-emphasis flips and brief reframings, not novel content.

`Candidate.sampler` records `cit_temperature`, `K_eff`, and `K_runtime`.

### 4.2 cascade — FE budget gate + apoha trajectory wiring

```python
ledger = FreeEnergyBudget(...)
draft, draft_audit = generate_draft(...)
ledger.earn_jnana(draft_audit["delta_F"])
ledger.earn_tokens(draft_audit["token_count"])
ledger.earn_aspect(draft_audit["aspect_coverage"])

apoha_trajectory = apohana_trajectory_from(draft_audit)  # NEW
brief = vimarsa(draft, aspects,
                evidence={"delta_F_draft": draft_audit["delta_F"]},
                iccha_apoha_trajectory=apoha_trajectory)

if not ledger.should_continue_revision():            # NEW gate
    state.committed = "draft"
    state.audit["revision_skipped_reason"] = "fe_budget_underwater"
    return state

shadow_rev, rev_audit = generate_revision(...)
state.surface_revision = shadow_rev
features = build_policy_features(draft_audit, rev_audit, vimarsa_event, ledger)
decision = commit_policy(state, features=features)   # NEW
state.committed = "revision" if decision else "draft"
```

### 4.3 vimarsa — receives `iccha_apoha_trajectory`

`run_cascade` constructs a per-candidate aspect-coverage trajectory from the `apohana` priors and passes it to `vimarsa.commit_decision(...)`. Items with no aspect dictionary still pass `switching_ok=True` via the existing fallback.

### 4.4 commit policies

`src/pce/policies/commit.py` defines:

```python
class CommitPolicy(ABC):
    name: str
    @abstractmethod
    def decide(self, state, features: PolicyFeatures, vimarsa_event: bool) -> bool: ...

class EventGated(CommitPolicy):
    def decide(self, state, features, vimarsa_event): return vimarsa_event

class AlwaysRevise(CommitPolicy):
    def decide(self, state, features, vimarsa_event): return True

class AlwaysDraft(CommitPolicy):
    def decide(self, state, features, vimarsa_event): return False

class LearnedGate(CommitPolicy):
    def __init__(self, model_path: Path | None = None): ...
    def decide(self, state, features, vimarsa_event):
        return self._model.predict_proba([features.as_vector()])[0, 1] > self.threshold

class OracleCommit(CommitPolicy):  # offline-analysis only
    def decide(self, state, features, vimarsa_event):
        return features.score_revision > features.score_draft
```

`LearnedGate` is trained by `scripts/train_learned_gate.py` on `audit/v0_3_traces/*.jsonl` with leave-one-domain-out CV. The trained sklearn model is serialized to `artifacts/learned_gate_v0_4.joblib` and committed to the repo (small file).

### 4.5 HaikuLM — rate-limit-aware error surface

`_call_cli_once` rewritten to:

```python
proc = subprocess.run(...)
stdout_text = proc.stdout.decode()
parsed: dict | None = try_parse_json(stdout_text)
if parsed and parsed.get("is_error") and parsed.get("api_error_status") == 429:
    raise HaikuRateLimitError(parsed.get("result", "rate-limited"), parsed)
if proc.returncode != 0:
    if parsed and parsed.get("is_error"):
        raise HaikuApiError(parsed.get("result", "cli error"), parsed)
    raise HaikuCLIError(f"rc={proc.returncode}", stderr=proc.stderr.decode())
return parsed
```

`HaikuRateLimitError` and `HaikuApiError` live in `src/pce/substrate/errors.py`.

- H8a.v4: paired (`shadow_revision` − `draft`) over all cascade items.

## 8. Risk register (deltas from v0.3)

| Risk | Mitigation |
|------|-----------|
| Best-of-K width inflates pilot cost above $30 | `K_max = 16` cap; IntegrityProbe ledger pre-flight estimates spend before the run; abort if projected > $30. |
| `LearnedGate` overfits the v0.3 training set | Leave-one-domain-out CV reported alongside in-distribution scores; ADR-002 mandates honesty. |
| Sonnet quota exhaustion mid-pilot | Pilot judge stage runs *after* the main pilot so partial loss is recoverable; bridge dry-run of 4 items required before full subset. |
| Haiku CLI rate-limit hits during the powered pilot | `HaikuRateLimitError` raises cleanly; driver records partial state and exits 1; user retries on the next quota window. |
| `cit_temperature` best-of-K mechanism does not actually diversify Haiku output | Phase-2 prove-gate measures n-gram entropy across candidates as a function of `cit_temperature`; if monotonicity fails, the v0.4 paper restates `cit_temperature` as recorded-only and demotes its claim. |
| Cascade still loses on the proxy but wins on the judge | Reported honestly. The contribution becomes "construct-validity warning for proxy composites" rather than a creativity claim. |
| Commit-policy multiplex introduces leakage between policies | All policies score the same draft/revision artifacts post-hoc; `LearnedGate` model never sees v0.4 evaluation data; calibration set is v0.3 traces only. |
